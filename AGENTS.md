# AGENTS.md

这个仓库（`llm-wiki` 分支）是一个**个人知识库（LLM Wiki）**。你（AI）负责读资料、写知识库；用户负责找资料、提问题、把方向。知识库的页面会在两个地方被阅读：普通浏览器，以及微信**「GH HTML 查看器」小程序**——小程序用 `mp-html` 渲染 HTML，且只内置了 `style` 插件。

这个模式来自 [Karpathy 的 LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)，但有一个关键改动：**`wiki/` 里的页面是 HTML（用 mp-html-page 技能生成），不是 Karpathy 原版的 Markdown**，这样能直接在小程序里浏览。

## 三层结构

1. **`raw/` — 原始资料（只读）**
   用户收集的源文档：文章、笔记、PDF、图片、数据。**这是事实来源，你只读不改。** 不要编辑或删除 `raw/` 里的任何东西。

2. **`wiki/` — 知识库（你来维护）**
   你生成的一组**互相关联的 HTML 页面**：每条资料的摘要页、概念 / 实体 / 主题页、对比页、总览。这一层完全归你管——你创建页面、在新资料到来时更新它们、维护交叉引用、保持一致。**用户读，你写。**

3. **`AGENTS.md` — 这份 schema**
   告诉你知识库怎么组织、有哪些约定、各种操作的流程。和用户一起随时间打磨它。

## wiki/ 的页面怎么写

`wiki/` 下的每个页面都是**一个自包含的 `.html` 文件**，用 **[`mp-html-page` 技能](.claude/skills/mp-html-page/SKILL.md)** 生成，确保能在小程序里正常渲染。要点：

- 所有 CSS 写在 `<head>` 的 `<style>` 里，且放在被它修饰的内容**之前**。
- **只用字面值**——不要 `var()`、`@media`、`@keyframes`、`@font-face`、`@import`、`*`、`[attr]`、`:hover` 等伪类。
- 在包裹所有内容的**外层 `<div>`** 上设置 `background` + `color`（否则小程序会回退到默认的白底黑字）。
- 不要 `<script>`、`<iframe>`、`<canvas>`、表单、动画或过渡。
- 图片：用 `<img>` 配 `https://` 链接或 `raw/` 里的相对路径——**不要**用 `background-image: url()`。用系统字体栈。

参考 `wiki/index.html` 的结构和配色，保持整个知识库视觉一致。

### 建议的目录与命名

- `wiki/index.html` — **总览 / 目录**，打开知识库时的入口页，列出所有页面 + 一句话摘要 + 路径。每次 ingest 都更新。
- `wiki/log.html` — **时间线日志**，只追加，记录每次 ingest / query / lint。
- `wiki/sources/<slug>.html` — 每条原始资料对应一个**摘要页**。
- `wiki/topics/<slug>.html` — 跨多条资料综合出来的**概念 / 实体 / 主题页**。

文件名用简短英文 slug，例如 `wiki/sources/karpathy-llm-wiki.html`。

### 交叉引用（小程序的特殊约定）

小程序一次只按路径打开一个文件，**页面之间的 `<a>` 链接在小程序里不能跳转**。所以：

- 在每个页面底部放一个「相关页面」区块，用**文字列出相关页面的路径**（例如 `wiki/topics/rag.html`），用户回到查看器手动打开。
- `wiki/index.html` 是导航中枢——保证每个页面都能从索引找到，避免没有入链的「孤岛」页。

## 操作

### Ingest（录入新资料）

用户往 `raw/` 放一条新资料并让你处理时：

1. 读这条资料，和用户聊聊关键要点。
2. 在 `wiki/sources/` 写一个摘要页（标题、来源、日期、要点、引用）。
3. 更新 `wiki/index.html`（加上新页面的条目，刷新计数与日期）。
4. 更新相关的概念 / 主题页（`wiki/topics/`）——补充新信息、加交叉引用、标注与旧说法矛盾的地方。一条资料可能牵动好几个页面。
5. 在 `wiki/log.html` 顶部追加一条记录。

### Query（提问）

用户对知识库提问时：

1. 先读 `wiki/index.html` 找相关页面，再读进去。
2. 综合出带**引用**的回答（指明信息来自哪个页面 / 哪条 `raw/` 资料）。
3. 好的回答可以**沉淀回知识库**：存成新的主题页或对比页，并更新索引和日志，让探索像录入一样累积。

### Lint（体检）

用户让你检查知识库时，找这些问题：页面之间的矛盾、被新资料推翻的旧说法、没有入链的孤岛页、被提到却没有独立页面的重要概念、缺失的交叉引用、可以补的资料缺口。提出要新增的页面和值得追的新问题。

## 索引与日志

- **`wiki/index.html`** 是**内容导向**的目录：每个页面一个条目，附一句话摘要，可选元数据（日期、来源数）。按类别组织（资料 / 概念 / 主题）。**每次 ingest 都更新。** 回答 query 时先读它来定位。
- **`wiki/log.html`** 是**时间导向**的只追加记录。每条以一致的前缀开头，例如 `[2026-06-20] ingest · 文章标题`，方便扫时间线。

## 提交

每次改完知识库，**commit 并 push** 到 GitHub，这样小程序才能打开最新内容。
