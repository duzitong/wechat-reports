数学界有个开了 86 年的著名猜想——雅可比猜想：只要一个「多项式映射」的雅可比行列式是个非零常数，它就一定可逆。

2026 年 7 月，一位数学家在 AI（Claude Fable 5）协助下给出了一个具体的三维反例：一个映射 ℂ³→ℂ³，雅可比行列式恒等于 −2（满足条件），却把三个不同的点映到了同一个点（因此不可逆）。条件成立、结论失败 = 反例。

妙处在于：验证它不需要读懂上百页证明——算一个行列式、代入几个点就行。只要会「代入」和「把其它字母当常数、求一次导」这一步（下面第一节就把它教给你），你就能自己核对。

一、先打地基（映射 · 导数 · 雅可比行列式）

多项式映射是什么？把它想成一台机器：塞几个数进去，吐几个数出来，而且吐出来的每个数都是用输入算出来的多项式（只有加、减、乘、乘方，没有除法开根）。比如一台「两进两出」的小机器 F(x, y) = (x + y, x · y)：喂 (2, 3) 进去，就得到 (2+3, 2×3) = (5, 6)。我们要研究的是三进三出的机器 F(x, y, z) = (f₁, f₂, f₃)，记作 F : ℂ³ → ℂ³。（用复数 ℂ 只是因为多项式在复数上最「干净」，你当成普通数字看就行。）

本文自始至终只用一个具体映射当例子——它也正是这次的反例。先把它的三个多项式 f₁、f₂、f₃ 原原本本写出来（长相有点唬人，但我们全程只会亲手动最简单的第三个 f₃；它是怎么冒出来的、可不可信，第二节再说）：

本文的主角 F：把 (x, y, z) 送到 (f₁, f₂, f₃) f₁ = (1 + xy)³ · z + y²(1 + xy)(4 + 3xy)
f₂ = y + 3x(1 + xy)² · z + 3x · y²(4 + 3xy)
f₃ = 2x − 3x²y − x³ · z　← 最简单，等下拿它练手

怎么判断这台机器是不是反例？关键看两个词：可逆（能不能由输出唯一地倒推回输入）和雅可比行列式。后者是把「每个输出对每个输入求偏导」排成一张 3×3 的表（雅可比矩阵），再压成一个数，用来量它在每一点「局部拉伸/翻转」了多少。听着绕，拆成三步就能自己搭出来 👇

第 1 步 · 先补两个词：导数 & 偏导数

导数（对某个字母「求导」）你可以先不管严格定义，只当成一条换项口诀：看一项「数字 × 字母的幂」，把指数搬到前面当系数、指数本身再减 1；纯数字（常数）没有字母，求导就变成 0；几项相加就各求各的、再加起来。

对 x 求导的口诀（换成别的字母同理） x² → 2x　（指数 2 搬下来当系数，2 − 1 = 1）
x³ → 3x²
2x → 2　（2x 就是 2x¹，指数 1 搬下来 ×2，剩 x⁰ = 1）
5 → 0　（常数里没有 x）

偏导数就是同一条口诀，只是现在字母多了：只对你盯的那个字母用口诀，其它字母全程当成普通数字（常数）。记号 ∂f₃/∂x 读作「f₃ 对 x 的偏导」。举一项体会一下：−3x²y 只盯 x —— 把 y 当数字，x² 用口诀变 2x，于是 −3 · (2x) · y = −6xy。

用这条口诀，把最简单的 f₃ = 2x − 3x²y − x³z 对三个字母各求一次偏导：

对每个字母各求一次（把另外两个字母当常数） 只盯 x：2x→2，−3x²y→−6xy，−x³z→−3x²z　⟹　∂f₃/∂x = 2 − 6xy − 3x²z
只盯 y：只有 −3x²y 含 y，其余都当常数　⟹　∂f₃/∂y = −3x²
只盯 z：只有 −x³z 含 z，其余都当常数　⟹　∂f₃/∂z = −x³

第 2 步 · 摆成矩阵（这就是「谁生成谁」）——每个输出都求出对 x、对 y、对 z 三个偏导。规则很死板：一个输出占一行，一个输入占一列，第 i 行第 j 列就填「第 i 个输出对第 j 个输入的偏导」。f₃ 刚算的三个，正好铺进最后一行：

输出 f₃
对 x 求导 → ∂f₃/∂x = 2 − 6xy − 3x²z → 进「对 x」列
对 y 求导 → ∂f₃/∂y = −3x² → 进「对 y」列
对 z 求导 → ∂f₃/∂z = −x³ → 进「对 z」列
行＝哪个输出　列＝对谁求导对 x对 y对 z
f₁∂f₁/∂x∂f₁/∂y∂f₁/∂z
f₂∂f₂/∂x∂f₂/∂y∂f₂/∂z
f₃2 − 6xy − 3x²z−3x²−x³

f₁、f₂ 两行同样是各求 3 个偏导，只是式子更长，这里用记号 ∂f₁/∂x… 占位；绿色那行就是我们上面亲手填出来的。这张 3×3 表就是雅可比矩阵。

第 3 步 · 压成一个数 = 行列式——行列式是把一张方阵算成一个数的固定口诀，照套即可。最简单的 2×2：[[a, b], [c, d]] 的行列式就是 a·d − b·c（主对角线之积 减 副对角线之积）。

3×3 不用背新口诀，而是沿第一行拆成三个 2×2：第一行每个元素，乘上「把它所在的那一行、那一列都划掉后剩下的 2×2 行列式」，符号按 + − + 交替，再加起来：

3×3 = 沿第一行拆成三个 2×2，符号 + − + 交替 [ a　b　c ]
[ d　e　f ]
[ g　h　i ]
det ＝
   + a·(ei − fh)
   − b·(di − fg)
   + c·(dh − eg)

括号里三个都是刚学的 2×2：划掉 a 的那行那列，剩 [[e, f], [h, i]] → ei − fh；划掉 b 剩 [[d, f], [g, i]] → di − fg；划掉 c 剩 [[d, e], [g, h]] → dh − eg。4×4、5×5 也是照这样一层层往下拆，所以矩阵再大也只是「长」，不是「难」。

回到我们的雅可比矩阵：把那 9 个格子按这个套路展开、化简后，对这个精心构造的 F，带 x、y、z 的项会全部两两抵消，只剩一个常数——这个常数就是 det(JF)（具体等于几，第三节亲手验）。

一个关键事实（必要条件）

如果 F 真的可逆、而且逆映射也是多项式，那么用链式法则一推，det(JF) 必然是一个非零常数（不含 x,y,z）。这一半是早就证明了的。

1939 年，Ott-Heinrich Keller 反过来问：这个必要条件够不够充分？也就是——

雅可比猜想（Keller, 1939） 若多项式映射 F : ℂⁿ → ℂⁿ 满足 det(JF) = 非零常数，
则 F 一定是可逆的（而且逆也是多项式）。

它看着「显然」，却是数学里出了名的硬骨头，开放了 86 年，期间倒下过无数个错误证明和错误反例。它还和一大堆其它问题等价，被认为是理解「多项式自同构」的核心关卡之一。

小背景：实数版早就被推翻了

如果把条件放宽成「雅可比行列式处处非零、但不必是常数」，并在实数上讨论，Pinchuk 在 1994 年就给过二维反例。但那是更弱的实数版；这次针对的是「雅可比为非零常数、在复数上」的原版猜想——难度不是一回事。

二、2026 年 7 月发生了什么

据多家报道：哈佛数学家 Levent Alpöge 在 Anthropic 的 AI 模型 Claude Fable 5 协助下，于 7 月 19–20 日（世界杯决赛期间）在推特上非正式地公布了一个针对雅可比猜想的显式反例，随后被 New Scientist 等转载、并被不少人用 Wolfram Alpha 等工具当场复算。

先把话说清楚（重要）：这则消息目前是推特上的非正式公布，尚未经过正式发表和同行评审。历史上「推翻雅可比猜想」的乌龙不止一次，所以「新闻本身」值得谨慎。

但——反例和「证明」不同：一个反例是不是对的，不取决于你信不信作者，只取决于那个具体的映射算出来对不对。而这恰恰是几分钟就能核对的事。下面我们不谈八卦，只看数学。

三、为什么它就是反例（两步验证）

「反例」的逻辑很简单：雅可比猜想说「满足条件 ⟹ 一定可逆」。要推翻它，只要找到一个映射既满足条件、又偏偏不可逆——要验的就是第一节写出的那个 F。于是只需验两件事：

验证 ①雅可比行列式是非零常数吗？

把上面三个多项式对 x, y, z 求偏导、排成 3×3 矩阵、算行列式（矩阵怎么由映射生成、偏导怎么求，见第一节第 2 步的图）。这是一步纯机械的多项式运算，用手（或任何符号计算工具）都能做。结果是：

det(JF) = −2　（一个非零常数，里面没有 x、y、z）

✓ 条件满足。本文作者用计算机代数把行列式完全展开，确认它处处恒等于常数 −2。

验证 ②它可逆吗？——不，它是「三对一」

可逆（单射）意味着「不同的输入必给不同的输出」。可这个 F 偏偏把三个不同的点送到了同一个点 (0, 1, 0)：

P₁ = (−2, 1, 2)
P₂ = (0, 1, −4)
P₃ = (2, −1/2, 5/4)
三个不同的点 ⟶ 同一个 F ⟶ 落到同一处
F(P₁) = F(P₂) = F(P₃) = (0, 1, 0)

既然不同输入给了相同输出，F 就不是单射，自然没有逆映射。（事实上它对几乎每个目标点都有 3 个原像，是个「3 对 1」的映射。）

✗ 结论失败：F 不可逆。

条件成立（①）+ 结论失败（②）= 一个货真价实的反例，把 n ≥ 3 的雅可比猜想推翻了。（注：二维 n = 2 的情形仍然悬而未决。）

四、你可以亲手验证

不信？挑最简单的 P₂ = (0, 1, −4) 代进去，x = 0 会消掉一大半项：

分量代入 x=0, y=1, z=−4（注意 1+xy = 1）结果
f₁1³·(−4) + 1²·1·(4+0) = −4 + 40
f₂1 + 3·0·(…) + 3·0·(…) = 11
f₃2·0 − 3·0²·1 − 0³·(−4) = 00

得到 (0, 1, 0)。再花两分钟把整数点 P₁ = (−2, 1, 2)（此时 1+xy = −1）代进去，同样得到 (0, 1, 0)。两个不同的点、同一个像——非单射，实锤。

这正是「反例 vs 证明」的差别

一个证明可能长达上百页，同行评审要花几个月去挑逻辑漏洞；而一个反例是「自证的」——你不必读懂作者怎么想到它，只要把这个具体对象拿来算：行列式是不是非零常数、点是不是真撞在一起。两步都是有限步的算术，谁都能重算。这也是为什么它能在公布几小时内就被大量独立复核。

五、这次不一样：AI 找例子，推特发结果

除了数学本身，这件事问世的方式也很值得一说。传统上，这种级别的结果通常要走完：数学家苦攻数年 → 写成几十页论文 → 挂上 arXiv、投稿期刊 → 经过数月乃至数年的同行评审 → 才正式发表。而这次几乎倒过来——结果是借助 AI（Claude Fable 5）算出来的，第一时间用一条推特公布（据说还是世界杯决赛当口随手发的），几小时内就被全球网友用 Wolfram Alpha 等工具复算。发现方式和发布方式，都和「传统数学」很不一样。

AI 到底做了什么？得说准确：这不是「AI 独自证明了一个定理」。人类这一方——Alpöge 和最初给他出题的朋友——负责提出并框定问题、判断往哪个方向找、以及确认结果的意义；AI 干的是大规模的符号计算与搜索：在庞大的多项式空间里，真的把这样一个「行列式恰好是常数、却又不可逆」的具体映射构造并验算出来。这类活儿正好人脑吃力、机器擅长——几十年里不少顶尖数学家凭直觉都没逮到的显式反例，被「人给方向 + AI 做苦力」的组合抓住了。

为什么「发推特」这次问题不大

如果是一份证明，绕过评审直接发推特很冒险——上百页逻辑里藏个漏洞，一时半会谁也看不出。但这次是反例：对不对只取决于那个具体映射算出来对不对（见第三、四节），是有限步、可独立重算的算术。所以「AI 找到 + 一条推特 + 全网几小时复核」这套非常规流程，对这种自证型结果反而走得通。

为什么说它是个「时刻」：对「寻找一个具体对象」这类问题，AI + 计算机代数展示了在人类直觉多年未破处突破的潜力——这正是这件事被反复讨论的核心。

但也别神化：「AI 推翻百年猜想」是绝佳标题，可最关键的一步——问对问题、知道要找什么——仍来自人类；AI 的确切贡献也还在厘清。把它理解成一个强力的合作者与加速器，比「AI 自己想通了一切」更贴近事实。

六、该信到什么程度

数学层面：就上面这个具体映射而言，两项检查都成立（本文已独立重算）。作为一个「满足假设却不可逆」的对象，它是自洽的。

新闻层面：公布方式是非正式的推特帖、尚无正式论文与同行评审；AI 与人各自的贡献（见第五节）也还在厘清。谨慎的说法是：「如果流传的映射就是最终版本，那这个反例站得住」，但正式定稿仍值得等。

别急着改教材：二维情形仍然开放，整套理论的后续影响也要时间消化。把它当成「一个漂亮、可复核的数学事件」来学习，而不是「已尘埃落定的定论」。
